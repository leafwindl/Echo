# Echo 数据库说明

本目录用于存放本地 SQLite 数据库文件，默认文件为：

```text
data/echo_memory.db
```

数据库文件、WAL 文件和 SHM 文件不会提交到 Git；本说明文档可以提交。

## 表清单

### `users`

存储用户基础身份。

主要内容：
- 内部用户 ID。
- 微信 openid。
- 昵称、头像等用户资料预留字段。
- 创建和更新时间。

被调用模块：
- Auth 模块：登录成功后写入或更新用户。
- Chat 模块：写入消息前确保用户存在。
- Memory 模块：写入长期记忆前确保用户存在。

### `conversations`

存储用户会话。

主要内容：
- 会话 ID。
- 所属用户 ID。
- 会话标题预留字段。
- 会话滚动摘要。
- 摘要覆盖到的消息边界。
- 会话状态。
- 创建和更新时间。

被调用模块：
- Chat 模块：获取或创建当前活跃会话。
- Chat 模块：构建上下文时读取会话摘要。
- Chat 模块：消息写入后更新会话时间。
- Chat 模块：消息较多时更新滚动摘要。

### `chat_messages`

存储聊天原始消息。

主要内容：
- 所属用户 ID。
- 所属会话 ID。
- 消息角色：用户或 AI。
- 消息文本内容。
- 消息类型：文本、语音识别文本、语音回复文本等。
- 消息时间。

被调用模块：
- Chat 模块：每轮对话写入用户消息和 AI 回复。
- Chat 模块：构建上下文时读取近期历史。
- Chat 模块：生成会话摘要时读取未摘要的历史消息。
- Voice 模块：语音回复最终复用 Chat 模块写入消息。
- Memory 模块：长期记忆可记录来源消息 ID。

### `user_memories`

存储用户长期记忆。

主要内容：
- 记忆 ID。
- 所属用户 ID。
- 记忆类型。
- 记忆内容。
- 来源消息 ID。
- 置信度。
- 重要性。
- 状态：active、inactive、deleted。
- 创建、更新和过期时间。

被调用模块：
- Memory 模块：抽取、创建、更新、触碰、停用和删除长期记忆。
- Memory 模块：记忆管理接口读取和清空记忆。
- Chat 模块：构建上下文时读取长期记忆。

### `memory_embeddings`

存储长期记忆的向量索引。

主要内容：
- 记忆 ID。
- 所属用户 ID。
- embedding 模型名。
- embedding 向量。
- 向量存储 ID 预留字段。
- 创建和更新时间。

被调用模块：
- Memory 模块：新建或更新记忆后写入向量。
- Memory 模块：删除或清空记忆时删除向量。
- Memory 模块：为历史记忆补齐 embedding。
- Chat 模块：构建上下文时通过向量召回相关长期记忆。

### `background_jobs`

存储后台任务。

主要内容：
- 任务 ID。
- 任务类型。
- 任务状态。
- 任务 payload。
- 已尝试次数和最大尝试次数。
- 错误信息。
- 创建、更新、开始和结束时间。

被调用模块：
- Memory 模块：调度和恢复长期记忆抽取任务。
- main 启动流程：服务启动时恢复可继续执行的记忆抽取任务。

## 注意事项

- 默认数据库位置由 `Echo-backend/db/connection.py` 定义。
- 可以通过 `ECHO_DB_PATH` 环境变量覆盖数据库路径。
- 不要手动提交 `.db`、`.db-wal`、`.db-shm` 文件。
- 如果发现多个旧数据库文件，应先确认服务启动日志中的实际 DB 路径，再决定是否迁移或删除旧数据。
