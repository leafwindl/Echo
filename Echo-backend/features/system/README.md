# System 模块

## 1. 业务目标

承载系统级接口，目前主要提供健康检查，用于本地调试、部署探活和外部网关验证服务是否可用。

## 2. 核心业务流程

1. 应用启动时注册系统接口。
2. 客户端请求健康检查地址。
3. System 模块不访问数据库或外部服务，直接返回服务存活状态。
4. 调用方根据返回结果判断后端进程是否可访问。
